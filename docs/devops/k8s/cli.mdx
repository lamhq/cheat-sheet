# CLI commands

## Apply

Create or update objects defined in YAML file:
```sh
kubectl apply -f pod.yml -f deployment.yml
```


## Pod

List all Pods along with their labels:
```sh
kubectl get pod --show-labels
```

Describle Pod:
```sh
kubectl describe pod [pod-name]
```

Forward a port in container to local machine:
```sh
kubectl port-forward [pod-name] [external-port]:[container-port]
```

Run commands in Pods:
```sh
kubectl exec [pod-name] -- [command]
```

Get shell access to a container:
```sh
kubectl exec -it [pod-name] -- sh
```


## Deployment commands

Create/update a Deployment:
```sh
kubectl apply -f deployment.yml
```

List all Deployments and their labels:
```sh
kubectl get deployment --show-labels
```

Get all Deployments with a specific label:
```sh
kubectl get deployment -l app=nginx
```

Delete deployment:
```sh
kubectl delete deployment [deployment-name]
```

Scale the number of replicas:
```sh
kubectl scale deployment [deployment-name] --replicas=5
```


## Service

List services:
```sh
kubectl get service
```

```sh
kubectl get svc svc-test
```

## Node

List all the nodes in cluster:
```sh
kubectl get nodes
```

## EndpointSlice

```sh
kubectl get endpointslices
```

```sh
kubectl describe endpointslice svc-test-n7jg4
```

## Config

View your config:
```sh
kubectl config view
```

## Context

View current context:
```sh
kubectl config current-context
```

List all the contexts defined in your kubeconfig file:
```sh
kubectl config get-contexts
```

Switch the current Kubernetes context:
```sh
kubectl config use-context my-context
```


## Namespaces

List all namespaces:

```sh
kubectl get namespaces
```

Get information about a namespace:

```sh
kubectl describe ns [namespace]
```

List all Service objects in namespace:

```sh
kubectl get svc --namespace kube-system
```

Switch to a namespace:

```sh
kubectl config set-context --current --namespace [namespace]
```

Create a new Namespace:
```sh
kubectl create ns hydra
```
