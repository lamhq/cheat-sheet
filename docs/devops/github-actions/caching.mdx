# Caching

You can use the official [actions/cache](https://github.com/actions/cache) action to cache files or directories.

Example for caching Node.js dependencies using npm:
```yml {14}
name: Caching with npm
on: push
jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - name: Cache node modules
        id: cache-npm
        uses: actions/cache@v4
        with:
          # npm cache files are stored in `~/.npm` on Linux/macOS
          path: ~/.npm
          key: ${{ runner.os }}-node-${{ hashFiles('**/package-lock.json') }}
          restore-keys: |
            ${{ runner.os }}-node-

      - if: ${{ steps.cache-npm.outputs.cache-hit != 'true' }}
        name: List the state of node modules
        continue-on-error: true
        run: npm list

      - name: Install dependencies
        run: npm install
```
- `path`: Directory to cache (e.g., `~/.npm`, `vendor/bundle`, `.venv`).
- `key`: Unique identifier for the cache. Use file hashes to invalidate cache when dependencies change.
- `restore-keys`: Fallback keys to use if the exact cache key isn't found.
- `cache-hit`: Output of the cache action, a boolean value to indicate an exact match was found for the key.
