# Variables

## About

Commands that run in actions or workflow steps can create, read, and modify variables.

You can set your own custom variables or use the default environment variables.


## Default environment variables

Default environment variables are available to every step in a workflow. They give you information about the workflow, job, runner, and GitHub repository context without needing to define them yourself.

Here are some of the most commonly used ones:

| Variable | Description |
|----------|-------------|
| `GITHUB_REF` | Git ref that triggered the workflow (branch or tag) |
| `GITHUB_SHA` | Commit SHA that triggered the workflow |
| `GITHUB_REPOSITORY` | Owner/repo name (e.g., `octocat/Hello-World`) |
| `GITHUB_ACTOR` | The user or app that triggered the workflow |
| `GITHUB_WORKFLOW` | Name of the workflow being run |
| `GITHUB_EVENT_NAME` | Name of the event that triggered the workflow (e.g., `push`, `pull_request`) |
| `GITHUB_JOB` | Name of the current job |
| `GITHUB_ACTION` | Name of the current action |
| `RUNNER_OS` | Operating system of the runner (`Linux`, `Windows`, `macOS`) |
| `RUNNER_ARCH` | Architecture of the runner (`X64`, `ARM`) |
| `RUNNER_TEMP` | Path to a temporary directory on the runner |
| `GITHUB_RUN_ID` | Unique ID for the workflow run |
| `GITHUB_RUN_NUMBER` | Incremental number for each run of a workflow |

You can reference these variables in your workflow like this:

```yaml
jobs:
  print-env:
    runs-on: ubuntu-latest
    steps:
      - name: Show environment variables
        run: |
          echo "Workflow: $GITHUB_WORKFLOW"
          echo "Repository: $GITHUB_REPOSITORY"
          echo "Actor: $GITHUB_ACTOR"
          echo "Commit SHA: $GITHUB_SHA"
```


## Custom environment variables

You can set environment variables at job level using `env:` key.

```yaml
name: Example Workflow
on: push

jobs:
  build:
    runs-on: ubuntu-latest
    env:
      API_URL: https://api.example.com
    steps:
      - run: echo "API URL is $API_URL"
```

You can also define variables for a single step only:

```yaml
steps:
  - name: Step with custom env
    env:
      GREETING: Hello
    run: echo "$GREETING World"
```


## Passing values between steps

You can set an environment variable in one step and make it available to later steps:

```yml
jobs:
  demo:
    runs-on: ubuntu-latest
    steps:
      - name: Set dynamic env var
        run: echo "MY_VAR=dynamic_value" >> $GITHUB_ENV

      - name: Use dynamic env var
        run: echo "Value is $MY_VAR"
```
