# Structured output

Here is the shortened, tl;dr version of the guide, focusing entirely on the native approach and omitting the Tool Strategy section:

## Introduction

Structured output allows agents to return data in specific, predictable formats (JSON objects, Pydantic models, or dataclasses) instead of raw natural language. The data is validated and returned inside the `structured_response` key of the agent's state.


## Specifying Response Format

```python
def create_agent(
    ...
    response_format: Union[
        ToolStrategy[StructuredResponseT],
        ProviderStrategy[StructuredResponseT],
        type[StructuredResponseT],
        None,
    ]
)
```

You can pass your Pydantic schema directly into the `response_format` argument of `create_agent()`. Under the hood, this configures the model's native capability.

When a schema type is provided directly, LangChain automatically chooses:

- `ProviderStrategy` if the model and provider chosen supports native structured output (e.g. OpenAI, Anthropic (Claude), or xAI (Grok)).
- `ToolStrategy` for all other models.


## Provider Strategy

```python
from dotenv import load_dotenv
from pydantic import BaseModel, Field
from langchain.agents import create_agent
from langchain_openrouter import ChatOpenRouter

load_dotenv()

class ContactInfo(BaseModel):
    """Contact information for a person."""
    name: str = Field(description="The name of the person")
    email: str = Field(description="The email address of the person")
    phone: str = Field(description="The phone number of the person")

model = ChatOpenRouter(model="openai/gpt-oss-20b", temperature=0.7)

agent = create_agent(model, response_format=ContactInfo)  # Auto-selects ProviderStrategy

result = agent.invoke({
    "messages": [{"role": "user", "content": "Extract contact info from: John Doe, john@example.com, (555) 123-4567"}]
})

print(result["structured_response"])
# ContactInfo(name='John Doe', email='john@example.com', phone='(555) 123-4567')
```


## Using `bind_tools`

When you pass a Pydantic model to `.bind_tools()`, OpenAI's API automatically converts it into a JSON schema that tells the LLM what structure to return.

```python
from pydantic import BaseModel, Field
from langchain_openai import ChatOpenAI
from langchain_core.messages import HumanMessage

class Answer(BaseModel):
    answer: str = Field(description="250 word answer")
    queries: list[str] = Field(description="Search queries")

llm = ChatOpenAI(model="gpt-4o-mini")
chain = llm.bind_tools(tools=[Answer], tool_choice="Answer")

# To use the chain, you'd invoke it like:
result = chain.invoke({"messages": [HumanMessage(content="Your question here")]})
```

The schema tells the LLM:
- Required fields: `answer` (string), `queries` (array)
- Descriptions guide what content to provide
- LLM must follow the structure exactly
