# Agents

AI Agents in LangChain.

## Introduction

See [AI Agents](../agents.mdx) for an introduction to AI agents.

### Chains vs. Agents

Unlike basic chains with hard-coded steps, an agent dynamically decides which tools or steps are required to solve an open-ended problem. The LLM determines what to do next based on the shifting context of the task.

- **Chains (Deterministic):** The developer explicitly hard-codes the control flow. While an LLM may perform specific steps (e.g., text summarization), it does not control the application's direction.
- **Agents (Dynamic):** The control flow is fluid and managed iteratively by the LLM. The model selects an action, evaluates the outcome, and continues reasoning until the user's objective is met.

## Tools

Tools are external functions or integrations that let the model safely interact with environments outside its static training data.

Common implementations include:

- Executing API requests or web search queries.
- Querying or updating internal relational databases.
- Running isolated Python code to handle precise data processing or mathematics.

## The ReAct Architecture

The **ReAct** framework (Reasoning and Acting) is a foundational agent architecture that operates in a continuous loop:

- **Reasoning (Thought):** The agent analyzes the objective and context using structured reasoning like Chain-of-Thought.
- **Acting (Action):** The LLM decides to trigger an external tool with generated input arguments.
- **Observation:** The application runs the tool, feeds the output back into the agent's context, and repeats the loop until resolved.

```
       ┌────────────────────────┐
       │     User Objective     │
       └───────────┬────────────┘
                   ▼
     ┌──► 1. REASONING (Thought)
     │             │
     │             ▼
     │    2. ACTING (Tool Choice)
     │             │
     │             ▼
     └─── 3. OBSERVATION (Tool Output)

```

## Implementation

Modern stateful agents are built through **LangGraph**, which provides pre-built ReAct agents to handle state maintenance and tool invocation seamlessly.

```python title="main.py"
from dotenv import load_dotenv
from langchain.agents import create_agent
from langchain_openrouter import ChatOpenRouter
from langchain.tools import tool

load_dotenv()

# 1. Define a tool function with descriptive docstrings
@tool
def internet_search(query: str) -> str:
    """Useful for searching weather information in New York."""
    return f"The weather in New York is sunny with a high of 75°F and a low of 41°F."

tools = [internet_search]
model = ChatOpenRouter(model="openai/gpt-oss-20b", temperature=0.7)

# 2. Compile the stateful agent loop
agent_executor = create_agent(model, tools)

# 3. Execute the agent
response = agent_executor.invoke({
    "messages": [("user", "What is the weather in New York?")]
})

print(response["messages"][-1].content)
```
