# LCEL

Compose LangChain components together using LangChain Expression Language.


## The `Runnable` Protocol

In LCEL, every component—from Prompt Templates, Chat Models, LLMs, Output Parsers, Retrievers inherits from the base **`Runnable`** class. It allows them to connect with each other seamlessly.

Every `Runnable` automatically supports the following standard execution methods out of the box:

- **`invoke(input)` / `ainvoke(input)**`: Processes a single input (Synchronous / Asynchronous).
- **`batch(inputs)` / `abatch(inputs)**`: Processes a list of inputs in parallel, automatically optimized via an internal thread pool.
- **`stream(input)` / `astream(input)**`: Yields chunks of data as soon as they are generated by the model, reducing perceived latency.


## Pipe Syntax (`|`)

LCEL utilizes Python's bitwise OR operator (`|`) to pass data from one component to the next. The output of the left component automatically matches and becomes the input of the right component.

A standard, fundamental chain looks like this: **Prompt $\rightarrow$ Model $\rightarrow$ Output Parser**.

```python
from langchain_core.prompts import ChatPromptTemplate
from langchain_core.output_parsers import StrOutputParser
from langchain_openrouter import ChatOpenRouter

# 1. Initialize components (Each component is a Runnable)
prompt = ChatPromptTemplate.from_template("Write a short poem about {topic}")
model = ChatOpenRouter(model="nvidia/nemotron-3-nano-omni-30b-a3b-reasoning:free", temperature=0.7)
output_parser = StrOutputParser()

# 2. Construct the chain using LCEL syntax
chain = prompt | model | output_parser

# 3. Execute the chain using invoke
result = chain.invoke({"topic": "artificial intelligence"})
print(result)
```


## Parallel Execution

Run multiple tasks concurrently using a single input.

```python
from langchain_core.runnables import RunnableParallel

joke_chain = ChatPromptTemplate.from_template("Tell me a joke about {topic}") | model | output_parser
poem_chain = ChatPromptTemplate.from_template("Write a poem about {topic}") | model | output_parser

# Run both branches concurrently to optimize execution time
parallel_chain = RunnableParallel(joke=joke_chain, poem=poem_chain)

response = parallel_chain.invoke({"topic": "programmers"})
# The return value is a dictionary containing the outputs of all parallel branches.
print(response["joke"])
print(response["poem"])
```

## Embedding Custom Python Code

`RunnableLambda` converts any regular Python function into a valid `Runnable`, allowing you to transform or manipulate data dynamically within your piped pipeline.

```python
from langchain_core.runnables import RunnableLambda

def uppercase_filter(text: str) -> str:
    return text.upper()

# Convert the function to a Runnable and place it at the end of the chain
custom_chain = prompt | model | output_parser | RunnableLambda(uppercase_filter)

print(custom_chain.invoke({"topic": "coffee"}))
# The final output will be forced into UPPERCASE letters
```
