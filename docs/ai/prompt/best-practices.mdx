# Best Practices

Best practices in prompt engineering to craft the perfect prompt that gives you exactly what you want.

## Add Context

Context provides relevance and helps the model understand what you actually want.

Without context, the model guesses and may generate off-topic, irrelevant responses.

With context, the model knows exactly what situation it's working within.

**Examples**:
- ⚠️ "Generate interview questions" (vague)
- ✅ "Generate technical interview questions **for a senior DevOps engineer position in a tech startup with a fast-paced culture working in the cloud**"


## Set Clear Tasks

The more specific your prompt, the more targeted the response.

**Examples**:

- ⚠️ "Improve the user experience of this e-commerce website" (Ambiguous)
  - Unclear what aspect to improve
  - No success metrics
  - Many possible interpretations

- ✅ "Identify and address specific pain points in the user experience to increase customer satisfaction and sales conversion rates"
  - Specific goal (pain points)
  - Measurable outcomes (customer satisfaction, conversion rates)
  - Targeted, accurate response


## Start General, Then Specific

Begin your prompt by giving the AI a broad description of the goal or scenario, and then follow up by listing any specific requirements.

For example:

```md
Write a JavaScript function that tells me if a number is prime

The function should take an integer and return true if the integer is prime

The function should error if the input is not a positive integer
```


## Break Complex Tasks

Break large or complicated tasks into multiple small, simple tasks and complete them one by one.

For example, instead of asking AI to generate a word search puzzle, break the process down into smaller tasks:

- Write a function to generate a 10 by 10 grid of letters.
- Write a function to find all words in a grid of letters, given a list of valid words.
- Write a function that uses the previous functions to generate a 10 by 10 grid of letters that contains at least 10 words.
- Update the previous function to print the grid of letters and 10 random words from the grid.


## Give Examples

Use examples of input data, expected outputs, or implementation styles to help AI understand exactly what you want.

```md
Write a Go function that finds all dates in a string and returns them in an array. Dates can be formatted like:

* 05/02/24
* 05/02/2024
* 5/2/24
* 5/2/2024
* 05-02-24
* 05-02-2024
* 5-2-24
* 5-2-2024

Example:

findDates("I have a dentist appointment on 11/14/2023 and book club on 12-1-23")

Returns: ["11/14/2023", "12-1-23"]
```

## Iterate and Refine

**The Iteration Cycle:**
1. Write a prompt
2. Test and evaluate the output
3. Refine based on results
4. Repeat until you get the perfect prompt

**Benefits:**
- Each iteration improves the previous one
- Results become more aligned with your goals
- Eventually you have an optimized prompt that delivers exactly what you need
