# Graph Database

## Overview

Stores data as nodes (entities) and edges (relationships between entities).

Designed to efficiently store, query, traverse complex relationships.

Excellent for data where connections and relationships are as important as the data itself.

Graph databases are like document stores but are designed for graphs instead of document trees.

Use cases: social networks, recommendation engines, fraud detection, knowledge graphs.

Options: Neo4J, Amazon Neptune, Azure Cosmos DB (Gremlin API)
