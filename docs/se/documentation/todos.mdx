# Todos

A directory contains a collection of development tasks, from oldest to newest, including:
- Feature requests (`feat`)
- Change requests (`crq`)
- Bug reports (`bug`)

Imported from task management tools (e.g., Jira, Trello) or manually maintained.

Contains tasks to implement or implemented. Not a product backlog or sprint backlog

Task will be linked directly to issues, pull requests, and code commits.

Bug report will be linked to relevant test cases.

- **Change requests** → stored in backlog + knowledge base for approvals.  
- **Bug reports** → stored in bug tracking system.  
- **Feature requests** → stored in product backlog as user stories.  

| **Type** | **Format** | **Example** |
|----------|------------|-------------|
| Change Request | Change log entry + approval workflow | *Request to add multi-language support; approved by Product Owner.* |
| Bug Report | Bug ID, steps to reproduce, expected vs actual result | *Bug #123: Login fails when password contains special characters.* |
| Feature Request | User story + acceptance criteria | *As a user, I want dark mode so I can reduce eye strain.* |

- Link them to **sprints, releases, and test cases** for traceability.  
- Use **tags or categories** (e.g., “Bug,” “Feature,” “Change Request”) to distinguish them.
