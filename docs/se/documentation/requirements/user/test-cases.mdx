# Test Cases

## Overview
This document lists detailed test cases for validating system functionality. Each test case includes an ID, description, preconditions, steps, expected results, and actual results (to be filled during execution).

## Test Case Format
| Field            | Description                                                                 |
|------------------|-----------------------------------------------------------------------------|
| **Test Case ID** | Unique identifier (e.g., `TC-001`)                                          |
| **Title**        | Short description of the test case                                          |
| **Preconditions**| Setup required before executing the test                                    |
| **Steps**        | Ordered list of actions to perform                                          |
| **Expected Result** | Outcome if the system behaves correctly                                  |
| **Actual Result**   | Outcome observed during execution (filled by QA team)                   |
| **Status**       | Pass / Fail / Blocked                                                      |
