# Project Charter

Guidelines for writing a Project Charter document.

## Overview

A Project Charter is a formal document that authorizes a project, defines its objectives, scope, stakeholders, and overall direction. 

It serves as the foundation for project planning and execution, ensuring alignment between the project team and stakeholders.


## Document Structure

A Project Charter typically includes the following sections:

### Project Information

- Project name, sponsor, manager, start/end dates.
- Background or context.

### Purpose and Objectives

- Why the project is being undertaken.
- Clear, measurable objectives.

### Scope

- What is included (deliverables, boundaries).
- What is excluded (out-of-scope items).

### Stakeholders

- Key stakeholders and their roles.
- Communication expectations.

### Resources and Budget

- High-level resource needs (people, tools).
- Estimated budget or funding source.

### Risks and Assumptions

- Known risks and constraints.
- Assumptions guiding the project.

## Best Practices

- **Concise and clear:** Keep it short (2–3 pages max).  
- **Business-focused:** Avoid technical details; focus on vision and goals.  
- **Accessible language:** Write for executives and non-technical stakeholders.  
- **Living document:** Update when project scope or objectives change.  

## Example

```md title="project-charter.md"
# Project Charter

## Project Information

- Project Name: Customer Support Portal Upgrade  
- Project Sponsor: Jane Smith, VP of Operations  
- Project Manager: John Doe  
- Start Date: April 1, 2026  
- End Date: September 30, 2026  

## Purpose and Objectives

- Background: Current support portal lacks self-service features, leading to high call volumes.
- Purpose: Enhance customer experience by reducing support wait times.  
- Objectives:  
  - Implement self-service knowledge base.  
  - Integrate chatbot for common queries.  
  - Reduce call center volume by 25% within six months of launch.

## Scope
- In-Scope: Portal redesign, chatbot integration, knowledge base creation.  
- Out-of-Scope: Call center staffing changes, CRM system upgrades.

## Stakeholders
- Sponsor: Jane Smith  
- Project Manager: John Doe  
- IT Team: Development and testing  
- Customer Service Team: Content creation and feedback

## Resources and Budget
- Estimated Budget: $250,000  
- Resources: 5 developers, 2 UX designers, 3 content writers

## Risks and Assumptions
- Risks: Delays in content creation, integration challenges with chatbot.  
- Assumptions: Funding approved, IT resources available as planned.
```
