# User Interfaces

## Stream-Aligned Teams

We decompose the system into microservices but leave a monolithic user interface. This leads having separate frontend and backend teams, slowing development due to the need to constantly coordinate changes and hand off work between teams.

Instead, we want stream-aligned teams, where  a team owning the end-to-end delivery of functionality in a specific part of the domain. A
team with end-to-end responsibility for delivery of user-facing functionality is also going to have a more obvious, direct connection to the end user as well. They can be more focused on making sure that the people they are serving are getting what they need.

The downsides of “backend” teams is they come with a hazy idea of what the software does or what the
users need, which can cause all sorts of misunderstandings when it comes to implementing new functionality.

To make that change happen and get rid of siloed frontend and backend teams, we need to break apart our user interfaces.


## Why Dedicated Frontend Teams Exist?

- **Scarcity of Experts**: UI development requires skills in interaction design, graphics, and web/native app development, making specialists rare.  
- **Consistency**: A single team owning the UI ensures a uniform look and feel across applications.  
- **Technical Challenges**: Some systems have complex UI needs that require dedicated focus.
