# Repository Setup

A quick checklist to set up a fresh codebase from scratch.

## Node.js projects

1. Create project repository from [Turborepo](./tools/turborepo.mdx#creating-project) template
2. [Add applications and packages](./tools/turborepo.mdx#managing-packages) to project
3. Set up formatter: [Prettier](./tools/prettier.mdx)
4. Set up code linter: [ESLint](./tools/eslint.mdx)
5. Set up commit message linter: [commitlint](./tools/commitlint.mdx)
6. Set up [git-branch-is](./tools/git-branch-is.mdx) to check branch name format
7. Set up [lint-staged](./tools/lint-staged.mdx) to validate git staged files
8. Set up git hooks with [Husky](./tools/husky.mdx)
9. Create CI pipeline to run tests, linting, and type checking
10. Create CD pipeline for automated release and deployment.
11. Create `README.md` file


## API projects

1. Set up API testing with [Jest](./tools/jest.mdx) and Supertest 
2. Configure ESLint to lint:
   1. TypeScript files
   2. Jest test files


## Web projects

1. Set test framework [Vitest](./tools/vitest.mdx) (if your project is using Vite)
2. Set up component testing with [Testing Library](../../web/react/react-testing-library.mdx)
3. Integrate UI documentation tool [Storybook](../../web/storybook.mdx)
4. Configure ESLint to lint:
   1. TypeScript React files
   2. Vitest test files
   3. Playwright test files
   4. Storybook files


## End-to-end testing project

1. Configure [Playwright](../../web/playwright.mdx)
2. Configure ESLint to lint Playwright test files
