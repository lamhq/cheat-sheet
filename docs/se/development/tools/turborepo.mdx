# Turborepo

## Version

This guide is for Turborepo v2.8.14 (March 2026).


## Command Snippets

### Creating project

Create an empty monorepo:

```sh
npx create-turbo@latest -e with-shell-commands
```

### Adding packages

Add a **NestJS** application package inside `apps/`:

```sh
npx turbo gen workspace --copy https://github.com/nestjs/typescript-starter --name <app-name> --destination apps/<app-name> --type app
```

Add a TypeScript Vite **React** application package:

```sh
npx turbo gen workspace --copy https://github.com/vitejs/vite/tree/main/packages/create-vite/template-react-ts --name <app-name> --destination apps/<app-name> --type app
```

**After creation**:
- Delete any lock files (`package-lock.json`, `yarn.lock`, `pnpm-lock.yaml`) in the generated package to prevent conflicts with the monorepo's lock file.  
- Update the `name` field in `package.json` to follow the convention:
  - Libraries → `@libs/{package-name}`  
  - Applications → `{app-name}`  
- If using [ESLint](./eslint.mdx#monorepo-setup):
  - Move ESLint dependencies to the root `package.json`.  
  - Merge the package's ESLint configuration into the root configuration.  


### Listing packages

Show all packages in your repository and where they're located:

```bash title="Terminal"
npx turbo ls
```

Show all packages that depend on a specific package:

```bash title="Terminal"
npx turbo ls -F ...<package-name>
```

### Listing tasks

```bash
npx turbo run
```

### Running tasks

Run a specific task for a package:

```bash
# run the `lint` task for the `web` package
npx turbo run web#lint
```

Running multiple tasks:

```bash
# run the `dev` tasks for `api` and `web` packages
npx turbo run web#dev api#dev

# Run the `build` task for the `docs` and `web` packages
npx turbo run build --filter=docs --filter=web
```

Run tasks for changed packages:

```bash
# Run any packages in `apps` subdirectories that have changed since the last commit
npx turbo run build --filter={.apps/*}[HEAD^1]
```

Exclude a package:

```bash
# Any packages in `apps` subdirectories except `./apps/admin`
npx turbo run build --filter=./apps/* --filter=!./apps/admin
```

Run task for package's dependents:

```bash
# run `lint` task for any packages that depend on `@acme/ui`
npx turbo run lint --filter=...@acme/ui
```

Run task for package's dependencies:

```bash
# run `build` task for `@acme/ui` and all of its dependencies
npx turbo run build --filter=@acme/ui...
```

Run tasks in watch mode:

```bash
# Run the `dev` and `lint` tasks in watch mode
npx turbo watch dev lint
```

## Set up npm scripts

For frequently used commands, you can add them as npm scripts in `package.json`:

```json title="package.json"
{
  "scripts": {
    "dev": "turbo watch dev",
    "build": "turbo run build",
    "lint": "turbo run lint",
    "test": "turbo run test"
  }
}
```

Then you can run them with `npm run dev`, `npm run build`, etc.


## Intgrating with Editor

Add `$schema` key in `turbo.json` so editor is able to provide full documentation and linting for invalid configuration:
```json title="turbo.json"
{
  "$schema": "node_modules/turbo/schema.json",
}
```
