# Turborepo

## Version

This guide is for Turborepo v2.8.14 (March 2026).


## Command Snippets

### Creating project

Create an empty monorepo:

```sh
npx create-turbo@latest -e with-shell-commands
```

### Adding packages

Add a NestJS application package inside `apps/`:

```sh
npx turbo gen workspace --copy https://github.com/nestjs/typescript-starter --name <app-name> --destination apps/<app-name> --type app
```

Add a TypeScript Vite React application package:

```sh
npx turbo gen workspace --copy https://github.com/vitejs/vite/tree/main/packages/create-vite/template-react-ts --name <app-name> --destination apps/<app-name> --type app
```

> [!NOTE]
> Remember to remove the lock files (`package-lock.json`, `yarn.lock`, `pnpm-lock.yaml`) from the generated package to avoid conflicts with your monorepo's lock file.

### Listing packages

Show all packages in your repository and where they're located:

```bash title="Terminal"
npx turbo ls
```

Show all packages that depend on a specific package:

```bash title="Terminal"
npx turbo ls -F ...<package-name>
```

### Managing dependencies

Turborepo does not manage dependencies. It's up to your package manager (PNPM, Yarn, or NPM).

This section list common commands to manage dependencies with PNPM.

Install a dependency to a project:
```bash
pnpm -F <project> add <package>
```

Install a local package to a project:
```bash
pnpm -F <project> add <package> --workspace
```

Install a dependency with a specific version range:
```bash
pnpm -F <project> add <package>@^1.0.0 --workspace
```

Install a dependency to the root of the monorepo:
```bash
pnpm add <package> -w
```

### Running npm scripts

Run a script of a specific project:

```bash
pnpm -F <project> run <script-name>
```

Run a script across **all projects** in the workspace:

```bash
pnpm -r run <script-name>
```

Run script in all projects whose name has prefix `@repo/`:
```bash
pnpm -F '@repo/**' run <script-name>
```

Run in projects that depend on `utils`:
```bash
pnpm -F '...utils' run <script-name>
```

Run a local installed binary from a specific project:

```bash
pnpm -F <project> exec <package>
```

### Listing tasks

```bash
npx turbo run
```

### Running tasks

Running multiple tasks:

```bash title="Terminal"
npx turbo run build test lint
```

Filtering by package:

```bash title="Terminal"
npx turbo -F @repo/api build
```

Filtering by directory:

```bash title="Terminal"
npx turbo -F @repo/api lint
```

Run task in a package and all packages that **depend on it**:

```bash title="Terminal"
npx turbo -F ...@repo/api lint
```

Run task in a package and all of **its dependencies**:

```bash title="Terminal"
npx turbo -F web... lint
```

Combining filters:

```bash title="Terminal"
npx turbo -F ...ui -F {./packages/*} -F [HEAD^1] build
```

Run tasks in watch mode:

```bash title="Terminal"
npx turbo watch dev lint
```

## Intgrating with Editor

Add `$schema` key in `turbo.json` so editor is able to provide full documentation and linting for invalid configuration:

```json title="turbo.json"
{
  "$schema": "node_modules/turbo/schema.json",
}
```
