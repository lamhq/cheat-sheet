# Prettier

## Version

This document is for version `3.8.1` (March 2026).


## Installation

<PackageManagerTabs command="install -D prettier@^3.8.1" />

Create a Prettier config file:

```ts title="prettier.config.mjs"
/**
 * @see https://prettier.io/docs/configuration
 * @type {import("prettier").Config}
 */
const config = {
  useTabs: false,
  tabWidth: 2,
  printWidth: 85,
  semi: true,
  singleQuote: true,
  trailingComma: 'all'
};

export default config;
```

> [!NOTE]
> Use a `.mjs` file instead of `.json` to allow reusing configuration across project. Also, Prettier run Node.js to load the config file, so it's better to use JavaScript over TyppeScript for compatibility.


## Using Prettier

Run Prettier:
```sh
npx prettier . --write
```

Check format without writing changes:
```sh
npx prettier . --check
```


## Enabling Organize Import

<PackageManagerTabs command="install -D prettier-plugin-organize-imports@^4.1.0" />

Add `plugins` section to Prettier config file:
```ts title="prettier.config.mjs"
/**
 * @see https://prettier.io/docs/configuration
 * @type {import("prettier").Config}
 */
const config = {
  // other options...
  plugins: ["prettier-plugin-organize-imports"]
};

export default config;
```


## IDE Integration

Install [Prettier - Code formatter](https://marketplace.visualstudio.com/items?itemName=esbenp.prettier-vscode) extension (`esbenp.prettier-vscode`).

Update VS Code settings to set Prettier as the default formatter:
```json title=".vscode/settings.json"
{
  "editor.defaultFormatter": "esbenp.prettier-vscode",
  "editor.formatOnSave": true
}
```

Add Prettier to VS Code Workspace Recommendations:
```json title=".vscode/extensions.json"
{
  "recommendations": [
    "esbenp.prettier-vscode"
  ]
}
```

## Ignoring code

```js
// prettier-ignore
matrix(
  1, 0, 0,
  0, 1, 0,
  0, 0, 1
)
```

Check [here](https://prettier.io/docs/ignore) for all available languages.

> [!NOTE]
> Prettier will follow rules specified in `.gitignore` if it exists the directory where Prettier is run.
