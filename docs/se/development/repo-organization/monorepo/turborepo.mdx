# Turborepo

## Introduction

Turborepo is a high-performance build system for JavaScript and TypeScript codebases.

It is designed for scaling monorepos.


## Benefits

Turborepo helps running tasks efficiently with:
- **Convenient Task Running**
  - Provides a standardized way to structure and run tasks across packages.
  - Makes local development faster by avoiding manual task running.
  - Works seamlessly with npm, Yarn, or pnpm workspaces.
- **Performance optimization**
  - Speeds up tasks (builds, tests, linting, etc.) by caching results and reusing them across runs.
  - Executes tasks in parallel and only re-runs what has changed.


## Limitations

🚫 Turborepo doesn't support:
- **Dependency management**: doesn't install or resolve dependencies; it relies on package manager.
- **Project management**: no code generation, scaffolding, or project setup features.
- **Architecture & design**: doesn't dictate architecture, testing strategy, or deployment models.
- **Versioning & publishing**: doesn't handle semantic versioning, changelogs, or publishing packages to registries (tools like Changesets or Lerna are needed).
- **Development workflow**: doesn't manage branching strategies, code reviews, or developer collaboration practices.
- **Non-JS/TS ecosystems**: primarily built for JavaScript/TypeScript; while adaptable, it's not a universal build tool for all languages.
- **CI/CD orchestration**: optimizes tasks but doesn't replace CI/CD platforms like GitHub Actions, GitLab CI, or Jenkins.
