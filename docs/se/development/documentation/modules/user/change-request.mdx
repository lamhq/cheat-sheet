# Change Request: Update Password Policy

## Overview
Modify the existing password policy to require stronger security standards.

## Current Behavior
- Passwords must be at least 8 characters.
- No requirement for special characters or numbers.
- No expiration policy.

## Requested Change
1. **Password Length**
   - Minimum length increased to 12 characters.

2. **Complexity**
   - Must include at least:
     - One uppercase letter
     - One lowercase letter
     - One number
     - One special character (!@#$%^&*)

3. **Expiration**
   - Passwords must expire every 90 days.
   - Users should be prompted to change their password upon expiration.

4. **Error Messages**
   - Provide clear feedback when password does not meet requirements.

## Acceptance Criteria
- [ ] User cannot set a password shorter than 12 characters.
- [ ] User cannot set a password without required complexity.
- [ ] System enforces 90‑day expiration and prompts user to update.
- [ ] Error messages display specific reason for rejection.

## Dependencies
- Authentication service (`AuthService.js`).
- Database schema (users table, password history).
- Frontend validation logic (registration and password reset forms).

## Testing Notes
- Unit tests for password validation logic.
- Integration tests for password reset and registration flows.
- End‑to‑end test for expiration prompt after 90 days.
