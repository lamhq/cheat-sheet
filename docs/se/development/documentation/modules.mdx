# Modules Overview

## Design principles

The system is organized into several logical modules.

Each module is responsible for a specific domain or functionality.

Each module is designed to be loosely coupled and independently deployable.  

Communication between services is primarily via gRPC (synchronous) or message queues (asynchronous).


## Main modules

- `user`: Handles user management, authentication, and authorization.
- `catalog`: Manages product catalog, inventory, and related operations.
- `order`: Responsible for order processing, payment, and fulfillment.
- `notification`: Manages sending notifications (email, SMS, push) to users.
- `payment`: Handles payment processing and integration with external payment gateways.
