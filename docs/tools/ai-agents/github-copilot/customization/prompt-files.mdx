# Prompt files

## Introduction

Prompt files (aka slash commands) are pre-defined prompts that you create for repetitive tasks. Each prompt file includes task-specific context and guidelines about how the task should be performed.

Prompt files are invoked manually in chat and stored in the `.github/prompts` folder or VS Code User profile.


## Use cases

Use prompt files when you want to:
- Create reusable prompts for common tasks in your project
- Add context for a specific chat request instead of all chats in a session (like custom instructions)
- Override default behavior of a custom agent

Example use cases:
- **Technical design**: design APIs, database schema, sequence diagrams, implementation plan base on requirements
- **Implement**: write code base on technical design, implementation plan
- **Writing tests**: design test cases base on requirements + technical design, implement unit/integration/end-to-end/API/UI tests
- **Code reviews**: check code follow standards, guidelines, patterns. detect security vulnerabilities or performance issues
- **Submitting PRs**: generate pull request titles, descriptions and submit them automatically
- **Debug test failures**: run test, capture and analyze test's output, provide suggestions on how to fix them
- **Troubleshooting production issues**: query production's log, investigate issues by analyzing the logs, provide suggestions on how to fix them
- **Writing documentation**: generate documentation from code, update documentation based on code changes
- **Communication**: explaining issues to non-technical stakeholders, generating status update messages, or drafting emails

> [!TIPS]
> You can provide additional instruction when invoking prompt files in chat.


## Prompt File Format

Prompt files are Markdown files and use the `.prompt.md` extension and have this structure:

### Header Optional

The header is formatted as YAML frontmatter with the following fields:

| Field | Purpose |
|-------|---------|
| **description** | Short summary of what the prompt does |
| **name** | Command name used after typing `/` in chat |
| **argument-hint** | Hint text shown in chat input |
| **agent** | Which Copilot agent to use (`ask`, `edit`, `agent`, or custom agent) |
| **model** | Which LLM model to run with |
| **tools** | List of tools available to this prompt |

```yaml
---
name: my-prompt
description: "Does something useful"
agent: ask
model: GPT-4
tools:
  - githubRepo
---
```

### Body

Contains the prompt text that is sent to the LLM.

Provide specific instructions, guidelines, or any other relevant information that you want the AI to follow.

You can reference:
- Other workspace files by using Markdown links
- Agent tools using `#tool:<name>`  
- Variables like:
  - `${workspaceFolder}`, `${workspaceFolderBasename}`
  - `${selection}`, `${selectedText}`
  - `${file}`, `${fileBasename}`, `${fileDirname}`, `${fileBasenameNoExtension}`
  - `${input:variableName}`, `${input:variableName:placeholder}` (pass values to the prompt from the chat input field)


## Create a prompt file

Ways to create a prompt file:
1. Type `/prompts` in the chat input to open the **Configure Prompt Files** menu.
2. Type `/create-prompt` in chat to generate a prompt file with AI.
3. Create a prompt from an ongoing conversation by asking `turn this into a reusable prompt` or `save this workflow as a prompt` after a multi-turn chat session.
4. Select Configure Chat (gear icon) > **Prompt Files**, and then select **New prompt** file:

![](https://code.visualstudio.com/assets/docs/copilot/customization/configure-chat-instructions.png)


## Manage prompt files

You can view and manage available prompt files by selecting `Chat: Configure Prompt Files` from the **Command Palette**.

To know where a prompt file comes from, hover over the prompt file in the list. The source location is displayed in a tooltip.


## Use a prompt file in chat

You have multiple options to run a prompt file:

1. In the **Chat view**, type `/` followed by the prompt name in the chat input field.

   You can add extra information in the chat input field. For example, `/create-react-form formName=MyForm` or `/create-api for listing customers`.

2. Run the `Chat: Run Prompt` command from the **Command Palette** and select a prompt file from the **Quick Pick**.

3. Open the prompt file in the editor, and press the play button in the title area. You can choose to run the prompt in the current chat session or open a new chat session.

> [!TIP]
> Use the `chat.promptFilesRecommendations` setting to show prompts as recommended actions when starting a new chat session.
