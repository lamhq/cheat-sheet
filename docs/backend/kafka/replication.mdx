# Replication

## Replication Factor

Each partition is replicated to other brokers.

A replication factor determines how many copies of a partition exist across the cluster.

In production, it's typically set to 2–3 (most commonly 3).

With a replication factor of 1 (development only), data loss occurs if a broker fails.


## Partition Leader

At any time only ONE broker can be a leader for a given partition

Producers can only send data to the broker that is leader of a partition. Consumers by default will read from the leader.

The other brokers will replicate the data

Each partition has one leader and multiple ISR.

**ISR (In-Sync Replica)**: A replica that is synchronized with the leader; if replication is fast enough, all replicas are ISRs


## Producer Acknowledgement

Producers can choose to receive acknowledgment of data writes:

- **acks=0**: Producer won't wait for acknowledgment (possible data loss, fastest)
- **acks=1**: Producer will wait for leader acknowledgment (limited data loss, moderate speed)
- **acks=all**: Leader + replicas acknowledgment (no data loss, slowest)


## Topic Durability

With a replication factor of **N**, you can permanently lose up to **N-1 brokers** and still have a complete copy of your data available in the cluster.

**Example:**
- Replication factor of 3 → can lose 2 brokers and still serve all data
- Replication factor of 1 → cannot lose any broker (no redundancy)
