# Customization

Customize the look and feel of your Rspress site.
  
## Home page

You can define the content of the Home page through frontmatter:

```yaml title="docs/index.mdx"
---
pageType: home
title: Rspress
titleSuffix: 'Rsbuild-based Static Site Generator'

hero:
  name: Rspress
  text: A documentation solution
  tagline: A modern documentation development technology stack
  actions:
    - theme: brand
      text: Introduction
      link: /en/guide/introduction
    - theme: alt
      text: Quick Start
      link: /en/guide/getting-started

features:
  - title: 'MDX Support'
    details: MDX is a powerful way to write content. You can use React components in Markdown.
    icon: 📦
  - title: 'Feature Rich'
    details: Out of box support for i18n, full-text search etc.
    icon: 🎨
  - title: 'Customizable'
    details: You can customize the theme ui and the build process.
    icon: 🚀
---
```

## Styling

### CSS variables

You can override [built-in CSS variables](https://rspress.rs/ui/vars) through importing your custom CSS file in `theme/index.tsx`:

```tsx title="theme/index.tsx"
import './index.css';
export * from '@rspress/core/theme-original';
```

Or use `globalStyles` in `rspress.config.ts`:

```ts title="rspress.config.ts"
import { defineConfig } from '@rspress/core';
import path from 'path';

export default defineConfig({
  globalStyles: path.join(__dirname, 'styles/index.css'),
});
```


### BEM classnames

Rspress uses BEM names for built-in components. Override styles by targeting these selectors:

```css title="styles/index.css"
.rp-[component-name]__[element-name]--[modifier-name] {
  /* styles */
}
```

Examples:

```css title="styles/index.css"
.rp-nav {}
.rp-link {}
.rp-tabs {}
.rp-codeblock {}
.rp-codeblock__title {}
.rp-codeblock__description {}
.rp-nav-menu__item,
.rp-nav-menu__item--active {}
```

## Components

Override built-in components with ESM re-exports.

The below example overrides the default `Layout` component to add some content before the navbar title, then re-exports everything from the original theme:

```tsx title="theme/index.tsx"
import { Layout as BasicLayout } from '@rspress/core/theme-original';

const Layout = () => <BasicLayout beforeNavTitle={<div>some content</div>} />;

export { Layout };
export * from '@rspress/core/theme-original';
```

To override the entire component, follow this [guide](https://rspress.rs/guide/basic/custom-theme#eject).

## Top-left logo

Configure the site logo with `logo` in `rspress.config.ts`:

```ts title="rspress.config.ts"
import { defineConfig } from '@rspress/core';

export default defineConfig({
  logo: 'https://avatars.githubusercontent.com/u/56892468?s=200&v=4',
});
```

`logo` supports:
- external URL
- absolute path (`/rspress-logo.png` from public)
- relative path (`./docs/public/rspress-logo.png`)

For dark mode, use object configuration of the logo:

```ts title="rspress.config.ts"
import { defineConfig } from '@rspress/core';

export default defineConfig({
  logo: {
    light: 'https://avatars.githubusercontent.com/u/56892468?s=200&v=4',
    dark: 'https://avatars.githubusercontent.com/u/56892468?s=200&v=4',
  },
});
```


## Favicon

Specify favicon with `icon`:

```ts title="rspress.config.ts"
import { defineConfig } from '@rspress/core';

export default defineConfig({
  icon: 'https://avatars.githubusercontent.com/u/56892468?s=200&v=4',
});
```

`icon` supports:
- external link
- absolute path (`/favicon.ico` from public)
- relative path (`./docs/public/favicon.ico`)
- `file://` protocol, such as `file:///local_path/favicon.ico`


## Homepage logo

In the frontmatter configuration of the homepage, you can specify the homepage logo image through the `hero.image.src` field

```md title="index.mdx"
---
pageType: home

hero:
  image:
    src: https://avatars.githubusercontent.com/u/56892468?s=200&v=4
    alt: Rspress
---
```

`hero.image.src` also supports:
- external URL
- absolute path from `public` folder


## Add global components

To add some custom global components to the page, use `globalUIComponents` in `rspress.config.ts`:

```ts title="rspress.config.ts"
import { defineConfig } from '@rspress/core';
import path from 'path';

export default defineConfig({
  globalUIComponents: [path.join(__dirname, 'components', 'MyComponent.tsx')],
});
```

You can also pass props:

```ts title="rspress.config.ts"
import { defineConfig } from '@rspress/core';

export default defineConfig({
  globalUIComponents: [
    [
      path.join(__dirname, 'components', 'MyComponent.tsx'),
      {
        foo: 'bar',
      },
    ],
  ],
});
```

Global components are automatically imported and rendered:

```tsx title="compSideEffect.tsx"
import { useEffect } from 'react';
import { useLocation } from '@rspress/core/runtime';

export default function PluginSideEffect() {
  const { pathname } = useLocation();

  useEffect(() => {
    // First-time mount
  }, []);

  useEffect(() => {
    // Route changes
  }, [pathname]);

  return null;
}
```


## Page layout

Use `pageType` in frontmatter to control layout.

In MDX page:

```md
---
pageType: custom
---
```

For Component page:

```tsx title="foo.tsx"
export const frontmatter = {
  pageType: 'custom',
};
```

Supported values:
- `home`: homepage layout
- `doc`: standard doc page with navbar, sidebar, outline
- `doc-wide`: wide doc layout
- `custom`: custom page with top navbar
- `blank`: custom page without top navbar
- `404`: not found page

Control individual layout elements via frontmatter:

- `navbar`: show/hide navbar
- `sidebar`: show/hide sidebar
- `outline`: show/hide outline
- `footer`: show/hide footer
- `globalUIComponents`: show/hide global components

Example:

```md title="foo.mdx"
---
navbar: false
sidebar: false
outline: false
footer: false
globalUIComponents: false
---
```

## Edit Link

Customize "Edit this page" link (in the doc footer and the right-side outline panel) to open the page in VS Code.

First, install `dotenv` as a dev dependency:

import { PackageManagerTabs } from '@theme';

<PackageManagerTabs command="install -D dotenv" />

Create a `.env` file with the path to your local `docs` folder:

```env title=".env"
REPO_BASE=vscode://file/Users/admin/Desktop/rspress/docs
```

Update `rspress.config.ts` to load the environment variable and configure `editLink`:

```ts title="rspress.config.ts"
import { defineConfig } from '@rspress/core';
import 'dotenv/config';

export default defineConfig({
  themeConfig: {
    editLink: {
      docRepoBaseUrl: process.env.REPO_BASE ?? '',
    },
  },
});
```
