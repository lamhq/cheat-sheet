# Navigation

## Introduction

Rspress can build navigation and sidebar automatically from:
- `_nav.json` for the top navbar
- `_meta.json` for sidebar structure

:::note
Automated navbar/sidebar will only work if there are no `nav` and `sidebar` configurations in the config file `rspress.config.ts`.
:::


Each navbar item is an object that follows this interface:

```ts
interface NavItem {
  // Navbar text
  text: string;
  // Navbar link
  link: '/';
  // Whether it is a download link
  download?: boolean;
  // Activation rules for navbar links
  activeMatch: '^/$|^/';
  // svg tag string or image URL(optional)
  tag?: string;
}
```

Example:

```json title="docs/_nav.json"
[
  {
    "text": "Guide",
    "link": "/guide/introduction",
    "activeMatch": "^/guide/"
  }
]
```

Refer to [nav config](https://rspress.rs/api/config/config-theme#nav) for details.


## Sidebar config

Sidebar is displayed on the left with multiple levels.

Sidebar items for a level are defined in `_meta.json` in each directory under `docs/`.

The order of items in `_meta.json` matches the order of items in the sidebar.

Sidebar items can be:
- **Link**: Corresponds to a file.
- **Collapsible menu**: Corresponds to a folder.
- **Section header**: Acts as a header only.
- **Divider**: Visually separates items.
- **Custom link**: Links to any URL, optionally with nested items.

Example directory:

```
docs
в”њв”Ђв”Ђ _nav.json
в””в”Ђв”Ђ example/
    в”њв”Ђв”Ђ _meta.json
    в”њв”Ђв”Ђ introduction.mdx
    в”њв”Ђв”Ђ advanced-topics/
    в”њв”Ђв”Ђ advanced-topics.mdx
    в””в”Ђв”Ђ other-topics/
```

```json title="docs/example/_meta.json"
[
  "introduction",
  {
    "name": "advanced-topics",
    "type": "dir",
    "collapsible": true,
    "collapsed": true
  },
  {
    "label": "Other Topics",
    "name": "other-topics",
    "type": "dir",
    "collapsible": true,
    "collapsed": true
  }
]
```

- `introduction.mdx`: Rendered as a link item in the sidebar. The label is automatically retrieved from the `h1` heading in the file.
- `advanced-topics/`: Rendered as a collapsible menu item because it is a folder. If a file with the same name exists, the folder takes precedence.
- `other-topics/`: Rendered as a collapsible menu item with an explicit `label` since no file exists to provide the label automatically.


## Tag icons

Sidebar and navbar items support `tag`, which can contain SVG or image URLs.

Example:

```json title="_meta.json"
{
  "type": "file",
  "name": "introduction",
  "label": "Introduction",
  "tag": "new, experimental"
}
```

You can also define tag through frontmatter in a page:

```mdx title="foo.mdx"
---
tag: new
---

# Foo

some text
```

Multiple tags are separated by commas:

```mdx
---
tag: new, experimental
---
```

You can also add tags in `_meta.json`:

```json title="_meta.json"
[
  {
    "name": "foo",
    "type": "file",
    "tag": "new, experimental"
  }
]
```

Rspress has built-in tags rendered as badges:

- `new`
- `experimental`
- `deprecated`
- `updated`


Use an SVG string directly:

```mdx
---
tag: <svg xmlns="http://www.w3.org/2000/svg" width="20" height="20" viewBox="0 0 24 24"><path fill="currentColor" d="M12 2L2 7l10 5 10-5-10-5z"/></svg>
---
```

Use an external image URL or data URL:

```mdx
---
tag: https://example.com/icon.png
---
```

```mdx
---
tag: data:image/svg+xml;base64,...
---
```

## JSON schema type hint

Rspress provides schema files for IDE support:
- `@rspress/core/meta-json-schema.json`
- `@rspress/core/nav-json-schema.json`

Example VS Code config:
```json title=".vscode/settings.json"
{
  "json.schemas": [
    {
      "fileMatch": ["**/_meta.json"],
      "url": "./node_modules/@rspress/core/meta-json-schema.json"
    },
    {
      "fileMatch": ["**/_nav.json"],
      "url": "./node_modules/@rspress/core/nav-json-schema.json"
    }
  ]
}
```
