# Navigation

## Introduction

Rspress can build navigation and sidebar automatically from:
- `_nav.json` for the top navbar
- `_meta.json` for sidebar structure

:::note
Automated navbar/sidebar will only work if there are no `nav` and `sidebar` configurations in the config file `rspress.config.ts`.
:::

## Basic usage

File placement:
- `docs/_nav.json` defines top-level navigation
- `docs/**/_meta.json` defines sidebar structure per directory

Example:
```text
docs
├── _nav.json
└── guide
    ├── _meta.json
    ├── introduction.mdx
    └── advanced
        ├── _meta.json
        └── plugin-development.md
```

If your document supports i18n, place `_nav.json` inside each locale folder:
```text
docs
├── en
│   ├── _nav.json
│   └── guide
│       ├── _meta.json
│       ├── introduction.mdx
│       └── advanced/_meta.json
└── zh
    ├── _nav.json
    └── guide
        ├── _meta.json
        ├── introduction.mdx
        └── advanced/_meta.json
```

## Navbar config

Refer to [nav config](https://rspress.rs/api/config/config-theme#nav) for details.

Each navbar item follows this interface:

```ts
interface NavItem {
  // Navbar text
  text: string;
  // Navbar link
  link: '/';
  // Whether it is a download link
  download?: boolean;
  // Activation rules for navbar links
  activeMatch: '^/$|^/';
  // svg tag string or image URL(optional)
  tag?: string;
}
```

Example:

```json title="docs/_nav.json"
[
  {
    "text": "Guide",
    "link": "/guide/introduction",
    "activeMatch": "^/guide/"
  }
]
```


## Sidebar config

A `_meta.json` file contains an array of sidebar items. Each item can be one of the following types:

```ts
export type FileSideMeta = {
  type: 'file';
  name: string;
  label?: string;
  tag?: string;
  overviewHeaders?: number[];
  context?: string;
};

export type DirSideMeta = {
  type: 'dir';
  name: string;
  label?: string;
  collapsible?: boolean;
  collapsed?: boolean;
  tag?: string;
  overviewHeaders?: number[];
  context?: string;
};

export type DirSectionHeaderSideMeta = Omit<DirSideMeta, 'type'> &
  Omit<SectionHeaderMeta, 'type'> & { type: 'dir-section-header' };

export type DividerSideMeta = {
  type: 'divider';
  dashed?: boolean;
};

export type SectionHeaderMeta = {
  type: 'section-header';
  label: string;
  tag?: string;
};

export type CustomLinkMeta =
  | {
      // file link
      type: 'custom-link';
      label: string;
      tag?: string;
      overviewHeaders?: number[];
      context?: string;
      link: string;
    }
  | {
      // dir link
      type: 'custom-link';
      label: string;
      tag?: string;
      overviewHeaders?: number[];
      context?: string;
      link?: string;
      collapsible?: boolean;
      collapsed?: boolean;
      items: _CustomLinkMetaWithoutTypeField[];
    };

export type SideMetaItem =
  | FileSideMeta
  | DirSideMeta
  | DirSectionHeaderSideMeta
  | DividerSideMeta
  | SectionHeaderMeta
  | CustomLinkMeta
  | string;
```

### string

When item is `string`, it means that the item is a file, and the file name is the string (without extension), for example:

```json title="_meta.json"
[
  "introduction",
  "advanced"
]
```

### file

When item is object with `type` of `file`, it create a sidebar item that corresponds to a file, where:

- `name` means the file name, without suffix
- `label` means the display name of the file in the sidebar

Example:

```json
{
  "type": "file",
  "name": "introduction",
  "label": "Introduction"
}
```

### dir

When item is object with `type` of `dir`, it create a sidebar item that corresponds to a directory, where:

- `name` means the directory name
- `label` means the display name of the directory in the sidebar
- `collapsible` means whether the directory can be collapsed
- `collapsed` means whether the directory is initially collapsed

Example:

```json
{
  "type": "dir",
  "name": "advanced",
  "label": "Advanced",
  "collapsible": true,
  "collapsed": false
}
```

### dir-section-header

Acts like a directory header in the UI:
```json
{
  "type": "dir-section-header",
  "name": "advanced",
  "label": "Advanced",
  "collapsible": true,
  "collapsed": false
}
```

### divider

```json
{
  "type": "divider",
  "dashed": true
}
```

### section-header

```json
{
  "type": "section-header",
  "label": "Section Header"
}
```

### custom-link

```json
{
  "type": "custom-link",
  "link": "/my-link",
  "label": "My Link"
}
```

Nested custom links:

```json
{
  "type": "custom-link",
  "label": "My Link",
  "items": [
    { "type": "custom-link", "label": "Sub Link 1", "link": "/sub-link-1" },
    { "type": "custom-link", "label": "Sub Link 2", "link": "/sub-link-2" }
  ]
}
```


## Tag icons

Sidebar and navbar items support `tag`, which can contain SVG or image URLs.

Example:

```json title="_meta.json"
{
  "type": "file",
  "name": "introduction",
  "label": "Introduction",
  "tag": "new, experimental"
}
```

You can also define tag through frontmatter in a page:

```mdx title="foo.mdx"
---
tag: new
---

# Foo

some text
```

Multiple tags are separated by commas:

```mdx
---
tag: new, experimental
---
```

You can also add tags in `_meta.json`:

```json title="_meta.json"
[
  {
    "name": "foo",
    "type": "file",
    "tag": "new, experimental"
  }
]
```

Rspress has built-in tags rendered as badges:

- `new`
- `experimental`
- `deprecated`
- `updated`


Use an SVG string directly:

```mdx
---
tag: <svg xmlns="http://www.w3.org/2000/svg" width="20" height="20" viewBox="0 0 24 24"><path fill="currentColor" d="M12 2L2 7l10 5 10-5-10-5z"/></svg>
---
```

Use an external image URL or data URL:

```mdx
---
tag: https://example.com/icon.png
---
```

```mdx
---
tag: data:image/svg+xml;base64,...
---
```

## JSON schema type hint

Rspress provides schema files for IDE support:
- `@rspress/core/meta-json-schema.json`
- `@rspress/core/nav-json-schema.json`

Example VS Code config:
```json title=".vscode/settings.json"
{
  "json.schemas": [
    {
      "fileMatch": ["**/_meta.json"],
      "url": "./node_modules/@rspress/core/meta-json-schema.json"
    },
    {
      "fileMatch": ["**/_nav.json"],
      "url": "./node_modules/@rspress/core/nav-json-schema.json"
    }
  ]
}
```
