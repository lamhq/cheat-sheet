# Installation

## Automatic installation

Create a new Next.js app in a new directory:
```sh
npx create-next-app@latest
```

## Run the development server

1. Run `npm run dev` to start the development server.
2. Visit http://localhost:3000 to view your application.
3. Edit the `app/page.tsx` file and save it to see the updated result in your browser.


## Set up TypeScript

### IDE

Next.js includes a custom TypeScript plugin and type checker, enable advanced type-checking and auto-completion.

To enable the plugin in VS Code:

1. Opening the **Command Palette**
2. Searching for `TypeScript: Select TypeScript Version`
3. Selecting `Use Workspace Version`


### Using Path Aliases

Next.js allows using absolute import paths, making it easier and cleaner to import modules.

```jsx
// Before
import { Button } from '../../../components/button'

// After
import { Button } from '@/components/button'
```

To configure absolute imports, add the `baseUrl` configuration option to your `tsconfig.json`:

```json title="tsconfig.json"
{
  "compilerOptions": {
    "baseUrl": "src/"
  }
}
```

You can use the `"paths"` option to `"alias"` module paths. For example, the following configuration maps `@/components/*` to `components/*`:

```json title="tsconfig.json"
{
  "compilerOptions": {
    "baseUrl": "src/",
    "paths": {
      "@/styles/*": ["styles/*"],
      "@/components/*": ["components/*"]
    }
  }
}
```

Each of the `"paths"` are relative to the `baseUrl` location.


## Set up ESLint

No setup required! Next.js automatically installs the necessary ESLint packages and configures the proper settings when running `create-next-app`.
