# Page Object Model

Using Page Object Model (POM) with Playwright.

## Introduction

Page Object Model (POM) is a **design pattern** in test automation that organizes web pages as objects, improving code readability, reusability, and maintainability by separating UI structure from test logic.

:::info
For example, an e-commerce web application might have a home page, a listings page and a checkout page. Each of them can be represented by page object models.
:::

## Example

Define a base page class to handle common operations:

```ts title="base.page.ts"
import { Locator, Page } from '@playwright/test';

/**
 * Base Page Object Model
 * Contains common methods shared across all page objects
 */
export abstract class BasePage {
  constructor(protected readonly page: Page) {}

  public abstract getUrl(): string;

  public navigate() {
    return this.page.goto(this.getUrl());
  }
}
```

Then, create a page object class for each page of your application:

```ts title="new-post.page.ts"
import { Locator, Page } from '@playwright/test';
import { BasePage } from './base.page';

export class NewPostPage extends BasePage {
  constructor(page: Page) {
    super(page);
  }

  public getUrl(): string {
    return '/new-post';
  }

  public getResultMessage(): Locator {
    return this.page.getByRole('alert');
  }

  public fillTitle(title: string): Promise<void> {
    return this.page.getByRole('textbox', { name: 'Title' }).fill(title);
  }

  public fillContent(content: string): Promise<void> {
    return this.page.getByRole('textbox', { name: 'Content' }).fill(content);
  }

  public submitForm(): Promise<void> {
    return this.page.locator('button:has-text("Submit")').click();
  }
}
```

Using the page object in tests:

```ts
import { expect, test } from '@playwright/test';
import { NewPostPage } from '../../pages/new-post.page';

test('should submit the form and show Success', async ({ page }) => {
  const examplePage = new NewPostPage(page);

  await examplePage.navigate();
  await examplePage.fillTitle('Post Title');
  await examplePage.fillContent('This is the content of the post.');
  await examplePage.submitForm();

  await expect(examplePage.getResultMessage()).toHaveText('Success');
});
```


## Best practices

### Return elements, not properties

Never return element properties (like text content, attribute values, counts, or visibility) in Page object methods:

Don't:

```ts
export class LoginPage {
  constructor(private page) {}

  async getResultMessage() {
    // ❌ Bad: returning element property
    return this.page.getByRole('alert').innerText();
  }
}

test('invalid credentials should show error', async ({ page }) => {
  const loginPage = new LoginPage(page);
  // ❌ Bad: get element property and perform non-retryable assertion
  const errorMessage = await loginPage.getResultMessage();
  expect(errorMessage).toBe('Invalid Username or password');
});
```

Do:

```ts
export class LoginPage {
  constructor(private page) {}

  async getResultMessage() {
    // ✅ Good: returning element directly
    return this.page.getByRole('alert');
  }
}

test('invalid credentials should show error', async ({ page }) => {
  const loginPage = new LoginPage(page);
  // ✅ Good: perform retryable assertion on the element
  await expect(loginPage.getResultMessage()).toHaveText('Invalid Username or password');
});
```

### Never perform assertions in methods

Never perform assertions inside methods of Page object.

Don't:

```ts
export class CheckoutPage {
  constructor(private page) {}

  async fillContent(content: string) {
    // ❌ Bad: assertion inside page object
    const emailField = this.page.getByLabel('Email');
    await expect(emailField).toBeVisible();
    await emailField.fill(content);
  }
}
```

Do:

```ts
export class CheckoutPage {
  constructor(private page) {}

  async fillContent(content: string) {
    // ✅ Correct: directly perform interactions without assertions in page object
    await this.page.getByLabel('Email').fill(content);
  }
}
```
