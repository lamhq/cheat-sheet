# Best Practices

Playwright Coding Best Practices.

## Test File Structure

Each test file should contain all test cases of a specific feature.

Name test cases using the pattern: `[action/condition] should [expected outcome]`:

```typescript title="login.spec.ts"
import { test, expect } from "@playwright/test";

// TC-01 (test case ID for traceability)
test("Login with valid credentials should redirect to dashboard", async ({ page }) => {
  // ... (Implementation)
});

// TC-02
test("Login with invalid credentials should display error message", async ({ page }) => {
  // ... (Implementation)
});

// TC-03
test("Submit empty form should show validation error", async ({ page }) => {
  // ... (Implementation)
});
```

Use `test.describe()` to group related test cases, only when there are more than one `describe` blocks in the file:

```typescript title="login.spec.ts"
import { test } from "@playwright/test";

test.describe("Login via Email", () => {
  // Test cases for email login...
});


test.describe("Login via Social Media", () => {
  // Test cases for social media login...
});
```

## Test case structure

Do not define a variable if it is only used once.

Use `test.step()` to break tests into logical phases, only when there're more than 3 steps and each step has more than 2 actions:

```typescript
import { test, expect } from '@playwright/test';

test('Checkout with a ATM card should create a card transaction', async ({ page }) => {

  await test.step('Create test data', async () => {
    // ... (Implementation)
  });

  await test.step('Initiate payment', async () => {
    // ... (Implementation)
  });

  await test.step('Pay', async () => {
    // ... (Implementation)
  });

  await test.step('Verify payment result', async () => {
    // ... (Implementation)
  });
});
```


## Finding elements

Use locators that refer to user-facing attributes (role, text and test id locators). It makes your tests resilient to changes in the DOM:

```ts
// 👍
page.getByRole('button', { name: 'submit' });
```

Do not use XPath or CSS selectors:

```ts
// 👎
page.locator('button.buttonIcon.episode-actions-later');
```

For elements that don't have natural roles or text, modify the HTML to add `data-testid` attribute for test stability:

```html
<div data-testid="cart-item-count">3</div>
```

```ts
page.getByTestId('cart-item-count')
```

Use chaining and filtering to narrow down to specific elements:

```ts
// 👍
await page
  .getByRole('listitem')
  .filter({ hasText: 'Product 2' })
  .getByRole('button', { name: 'Add to cart' })
  .click();
```

Avoid brittle CSS selectors:

```ts
// 👎
page.locator('.product-list > .product-item:nth-child(1) .add-to-cart-btn');
```


## Assertions

Use web-first assertions with automatic waiting:

```ts
// 👍
await expect(page.getByText('welcome')).toBeVisible();

await expect(page).toHaveURL('/dashboard');

await expect(page.getByRole('button', { name: 'Checkout' }))
  .toHaveText('Checkout');

await expect(page.getByTestId('cart-icon'))
  .toHaveAttribute('aria-label', 'Shopping cart');

await expect(page.getByTestId('date-header')).toHaveText('Date');
```

Do not retrieve element's content or attribute value and perform manual assertions on them:

```ts
// 👎
const descriptionText = await description.textContent();
expect(descriptionText).toMatch(
  /[A-Z][a-z]{2},\s+\d{1,2}\s+[A-Z][a-z]{2},\s+\d{4}/
);

expect(await page.getByText('welcome').isVisible()).toBe(true);

expect(page.url()).toContain('/dashboard');
```

Never perform assertions inside conditional blocks (e.g., `if`, `else`, `switch`):

```ts
// 👎
test('login form should show error on empty submit', async ({ page }) => {
  await page.goto('/login');
  await page.getByRole('button', { name: 'Sign in' }).click();

  // ❌ Bad: assertion inside conditional
  if (await page.locator('.error').isVisible()) {
    await expect(page.locator('.error')).toHaveText('Invalid Username or password');
  }
});
```


## Page Object Model

Use POM to encapsulate page behavior, interactions, and selectors:

```ts title="pages/login.page.ts"
import { BasePage } from './base.page';

export class LoginPage extends BasePage {

  public enterUsername(username: string) {
    return this.page
      .getByRole('textbox', { name: /Username or email/i })
      .fill(username);
  }

  public enterPassword(password: string) {
    return this.page
      .getByRole('textbox', { name: /Password/i })
      .fill(password);
  }

  public clickLoginButton() {
    return this.page
      .getByRole('button', { name: /Sign In/i })
      .click();
  }
}
```


Keep tests focused on behavior, not locator details:

```ts title="login.spec.ts"
import { test, expect } from '@playwright/test';
import { LoginPage } from '../pages/login.page';

test('Login with valid credentials should navigate to dashboard', async ({ page }) => {
  const loginPage = new LoginPage(page);
  await loginPage.goto();

  await loginPage.enterUsername('testuser');
  await loginPage.enterPassword('password123');
  await loginPage.clickLoginButton();

  await expect(page).toHaveURL('/dashboard');
});
```

Never return element properties (like text content, attribute values, counts, or visibility) in Page object methods:

```ts title="pages/login.page.ts"
// 👎
export class LoginPage {
  constructor(private page) {}

  async getErrorMessage() {
    // ❌ Bad: returning element property
    return await this.page.locator('.error').innerText();
  }
}

// test
test('invalid credentials should show error', async ({ page }) => {
  const loginPage = new LoginPage(page);

  const errorMessage = await loginPage.getErrorMessage();
  expect(errorMessage).toBe('Invalid Username or password');
});
```

Page object methods should never perform assertions:

```ts title="pages/checkout.page.ts"
// 👎
export class CheckoutPage {
  constructor(private page) {}

  async assertPaymentMethodIsCard() {
    // ❌ Bad: assertion inside page object
    await expect(this.page.getByTestId('payment-method-label')).toHaveText('Card');
  }
}

// test
test('checkout with ATM card should display payment method as Card', async ({ page }) => {
  const checkoutPage = new CheckoutPage(page);

  // ❌ Test delegates assertion to page object
  await checkoutPage.assertPaymentMethodIsCard();
});
```
