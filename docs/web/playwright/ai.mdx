# AI Agent

Integrate Playwright with AI agents.

## Test Agents

### Introduction

Test Agents are a set of of instructions and MCP tools for generating and fixing Playwright tests:
- `planner` creates a human-readable Markdown test plan
- `generator` turns test plan into executable Playwright test files
- `healer` repairs failing generated tests by updating selectors, waits, or test flow

Test Agents should be regenerated whenever Playwright is updated.

### Installation

Generate agent definitions for your project with:

```bash
npx playwright init-agents --loop=vscode
```

- Agent definitions are stored under `.github/agents`
- Regenerate them after updating Playwright to keep the tools and instructions current

### Planner

Planner agent explores the app and produces a test plan

Input:
- a clear request such as “Generate a plan for login functionality”
- optional Product Requirement Document for context

Output:
- A Markdown test plan. Stored in `specs/`


### Generator

Generator agent uses the Markdown plan to produce executable Playwright Tests.

It verifies selectors and assertions against the actual UI in real time.

Input:
- Test plan file

Output:
- A test suite under `tests/`
- generated tests may fail and can be healed automatically by the healer agent 


### Healer

Healer agent fixes failing tests.

It:
- replays failing steps
- inspects the UI for updated locators
- suggests patches (e.g., locator update, wait adjustment, data fix)
- reruns tests until they pass or stop safely

Input:
- failing test name

Output:
- a passing test, or a skipped test if the healer believes that functionality is broken.


## Playwright MCP

:::note
If you already installed Test Agents, it included MCP server installation as well, so you can skip this section.
:::
### Introduction

The Playwright MCP server provides browser automation capabilities, enabling LLMs to interact with web pages.


### Installation

To install the Playwright MCP server to VS Code via the VS Code CLI:

```bash
code --add-mcp '{"name":"playwright","command":"npx","args":["@playwright/mcp@latest"]}'
```

### Usage

Ask your AI assistant to interact with a web page:

```
Navigate to https://demo.playwright.dev/todomvc and add a few todo items.
```

Other capabilities include:
- Navigate to a page: ask "Go to https://example.com"
- Click an element: ask "Click the 'Submit' button"
- Fill a form: ask "Fill the 'Email' field with 'example@example.com'"
- Take a screenshot: ask "Take a screenshot of the page"
- Extract information: ask "What is the title of the page?" or "List all the links on the page"
- Run Playwright code: ask "Run this Playwright code: `await page.click('button#submit')`"
- Mock an API: ask "Mock the API endpoint '/api/data' to return `{ data: 'mocked data' }`"

### Configuration options

- [Run in headless mode](https://playwright.dev/docs/getting-started-mcp#headed-mode).
- [Choose browser to run](https://playwright.dev/docs/getting-started-mcp#browser-selection).
- [User profiles & browser extension](https://playwright.dev/docs/getting-started-mcp#user-profile)

## Playwright CLI

### Introduction

Playwright CLI is a command-line interface for running Playwright tests.

It is designed for coding agents. It provides token-efficient browser control and installable skills.

### Installation

Install globally:

<PackageManagerTabs command="install -g @playwright/cli" />

Or run locally via `npx`:

```bash
npx playwright-cli --help
```

Install skills:

```bash
npx playwright-cli install --skills
```

### Usage

Try asking your coding agent:

```
Use playwright skills to test https://demo.playwright.dev/todomvc/.
Take screenshots for all successful and failing scenarios.
```
