# Write tests

Code snippets for writing browser automation tests.

## Finding elements

[Locators](https://playwright.dev/docs/locators) are used to find elements on the page.

| Locator | Description | When to Use |
| --- | --- | --- |
| `getByRole()` | Locate by explicit and implicit accessibility attributes. | For buttons, links, menus. |
| `getByText()` | Locate by text content. | For headings, paragraphs. |
| `getByLabel()` | Locate a form control by associated label's text. | For inputs, checkboxes. |
| `getByPlaceholder()` | Locate an input by placeholder. | For unlabeled text fields. |
| `getByAltText()` | Locate an element, usually image, by its text alternative. | For images/media. |
| `getByTitle()` | Locate an element by its title attribute. | For tooltip info. |
| `getByTestId()` | Locate an element based on its `data-testid` attribute. | When no semantic locator available. |
| `locator()` | Use CSS or XPath selectors directly (last resort). | The last option when no other locator works. |

```ts
// Locate a button by role and accessible name
/* <button type="submit">Save changes</button> */
await page.getByRole('button', { name: 'Save changes' }).click();

// Locate a heading by role and name
/* <h1>Dashboard</h1> */
await page.getByRole('heading', { name: 'Dashboard' }).click();

// Locate a link by role and accessible name
/* <a href="/learn-more">Learn more</a> */
await page.getByRole('link', { name: 'Learn more' }).click();

// Locate a checkbox by role and label
/* <label><input type="checkbox"> Accept terms</label> */
await page.getByRole('checkbox', { name: 'Accept terms' }).check();

// Locate a radio button by role and label
/* <label><input type="radio" name="option"> Option A</label> */
await page.getByRole('radio', { name: 'Option A' }).check();

// Locate a textbox (input) by role and label
/* <label for="username">Username</label>
   <input id="username" type="text"> */
await page.getByRole('textbox', { name: 'Username' }).fill('lac');

// Locate a combobox (dropdown) by role and label
/* <label for="country">Country</label>
   <select id="country"><option value="VN">Vietnam</option></select> */
await page.getByRole('combobox', { name: 'Country' }).selectOption('VN');

// Locate a tab by role and name
/* <button role="tab">Settings</button> */
await page.getByRole('tab', { name: 'Settings' }).click();

// Locate a menuitem inside a popup menu
/* <ul role="menu">
     <li><button role="menuitem">Delete</button></li>
   </ul> */
await page.getByRole('menuitem', { name: 'Delete' }).click();

// Match visible text content
await page.getByText('Welcome back!').click();

// Target form controls by label text
await page.getByLabel('Email').fill('user@example.com');

// Locate inputs by placeholder text
await page.getByPlaceholder('Enter your password').fill('secret123');

// Find images by alt attribute
await page.getByAltText('Company logo').click();

// Match elements by title attribute
await page.getByTitle('Close').click();

// Use custom data-testid attributes
await page.getByTestId('login-button').click();

// Use CSS or XPath selectors directly (not recommended)
await page.locator('div.card > button.action').click();
```

### Filtering locators

Locators can be filtered to narrow down the matched elements using `locator.filter()`.

Filtering options:

- `hasText`: element contains a string or regex
- `hasNotText`: element does not contain a string
- `has`: element has a matching descendant locator
- `hasNot`: element does not have a matching descendant locator

```ts
// Filter by text
await page
  .getByRole('listitem')
  .filter({ hasText: 'Product 2' })
  .getByRole('button', { name: 'Add to cart' })
  .click();

// Filter by text using regex
await page
  .getByRole('listitem')
  .filter({ hasText: /Product 2/ })
  .getByRole('button', { name: 'Add to cart' })
  .click();

// Filter by not having text
await page
  .getByRole('listitem')
  .filter({ hasNotText: 'Out of stock' })
  .click();

// Filter by child/descendant and click the nested button
await page
  .getByRole('listitem')
  .filter({
    has: page.getByRole('heading', { name: 'Product 2' })
  })
  .getByRole('button', { name: 'Add to cart' })
  .click();

// Verify there is only one matching card
await page
  .getByRole('listitem')
  .filter({
    has: page.getByRole('heading', { name: 'Product 2' })
  })
  .click();

// Filter by not having a child/descendant with specific text
await page
  .getByRole('listitem')
  .filter({ hasNot: page.getByText('Product 2') })
  .click();

// Matching only visible elements
await page
  .locator('button')
  .filter({ visible: true })
  .click();

// Chain multiple filters to narrow down the selection
await page.getByRole('listitem')
  .filter({ hasText: 'Mary' })
  .filter({ has: page.getByRole('button', { name: 'Say goodbye' }) })
  .screenshot({ path: 'screenshot.png' });
```

### Combining locators

Matching two locators simultaneously (AND logic):

```ts
// <button title="Subscribe">Subscribe Now</button>
const button = page.getByRole('button').and(page.getByTitle('Subscribe'));
```

Matching one of two locators (OR logic):

```ts
const newEmail = page.getByRole('button', { name: 'New' });
const dialog = page.getByText('Confirm security settings');
await expect(newEmail.or(dialog).first()).toBeVisible();
```

### Nested locators

Locators can be chained to to narrow down to a particular part of the search.

```ts
// Within a dialog, finds a button with the accessible name "OK" and click it
await page
  .getByRole('dialog')
  .getByRole('button', { name: 'OK' })
  .click();
```

## Actions

### Element actions

```ts
// Check the input checkbox
await page.getByLabel('Accept terms').check();

// Click the element
await page.getByRole('button', { name: 'Submit order' }).click();

// Uncheck the input checkbox
await page.getByLabel('Subscribe to newsletter').uncheck();

// Hover mouse over the element
await page.getByRole('link', { name: 'More options' }).hover();

// Fill the form field, input text
await page.getByLabel('Username').fill('lac');

// Focus the element
await page.getByPlaceholder('Enter your email').focus();

// Press single key
await page.getByPlaceholder('Search products').press('Enter');

// Pick files to upload
await page.getByLabel('Upload resume').setInputFiles('resume.pdf');

// Select option in the drop down
await page.getByLabel('Country').selectOption('VN');
```

For the complete list, check the [Input API](https://playwright.dev/docs/input) section.


### Page actions

```ts
// Navigate to a specific URL
await page.goto('https://example.com');

// Take a screenshot of the page
await page.screenshot({ path: 'homepage.png' });

// Generate a PDF (Chromium only)
await page.pdf({ path: 'report.pdf', format: 'A4' });

// Run custom JavaScript in the page context
const title = await page.evaluate(() => document.title);
console.log(title);

// Inject scripts before any page loads
await page.addInitScript({ path: './preload.js' });

// Capture cookies and localStorage
const storage = await page.context().storageState({ path: 'storage.json' });
console.log(storage);
```

## Assertions

### Page-Level Assertions

- `toHaveTitle()` – page has expected title  
- `toHaveURL()` – page has expected URL
- `toHaveScreenshot()` – page has a screenshot  
- `toMatchAriaSnapshot()` – page matches ARIA snapshot

```ts
await expect(page).toHaveURL(/inventory.html/);
```

### Element State Assertions

```ts
// Element is visible
await expect(page.getByRole('button', { name: 'Submit' })).toBeVisible();

// Element is hidden or not in DOM
await expect(page.getByText('Loading...')).toBeHidden();

// Element is enabled
await expect(page.getByRole('button', { name: 'Save changes' })).toBeEnabled();

// Element is disabled
await expect(page.getByRole('button', { name: 'Delete' })).toBeDisabled();

// Checkbox or radio is checked
await expect(page.getByLabel('Accept terms')).toBeChecked();

// Input is editable
await expect(page.getByLabel('Username')).toBeEditable();

// Element has no text content
await expect(page.getByRole('heading', { name: '' })).toBeEmpty();

// Element is focused
await expect(page.getByPlaceholder('Enter your email')).toBeFocused();

// Element intersects viewport
await expect(page.getByRole('heading', { name: 'Dashboard' })).toBeInViewport();

// Element is attached
await expect(page.getByTestId('product-card')).toBeAttached();
```

### Content & Attribute Assertions

```ts
// Locator matches expected number of elements
await expect(page.getByRole('listitem')).toHaveCount(3);

// Element has exact text
await expect(page.getByRole('heading', { level: 1 })).toHaveText('Dashboard');

// Element contains partial text
await expect(page.getByText('Welcome')).toContainText('Welcome back');

// Element has specific attribute
await expect(page.getByRole('link', { name: 'Learn more' }))
  .toHaveAttribute('href', '/learn-more');

// Element has specified CSS classes
await expect(page.getByTestId('alert')).toContainClass(['error', 'visible']);

// Element has specified CSS class property
await expect(page.getByTestId('alert')).toHaveClass('error');

// Input has a specific value
await expect(page.getByLabel('Username')).toHaveValue('lac');

// Select has options selected
await expect(page.getByLabel('Country')).toHaveValues(['VN', 'US']);

// Element has a specific ARIA role
await expect(page.getByRole('button', { name: 'Save changes' })).toHaveRole('button');

// Element has an ID
await expect(page.getByRole('heading', { name: 'Dashboard' })).toHaveId('main-title');

// Element has specific CSS property
await expect(page.getByRole('button', { name: 'Submit' }))
  .toHaveCSS('background-color', 'rgb(0, 123, 255)');

// Element has a matching accessible description
await expect(page.getByRole('button', { name: 'Info' }))
  .toHaveAccessibleDescription('More information');

// Element has a matching accessible name
await expect(page.getByRole('button', { name: 'Save changes' }))
  .toHaveAccessibleName('Save changes');

// Element matches the Aria snapshot
await expect(page.getByRole('form')).toMatchAriaSnapshot(`
  - role: form
    name: Registration
    children:
      - role: textbox
        name: Username
      - role: button
        name: Submit
`);
```

### Response Assertions

```ts
// Response status is OK (200–299)
const response = await page.goto('https://example.com');
await expect(response).toBeOK();

// Response has specific status code
const apiResponse = await page.request.get('https://api.example.com/data');
await expect(apiResponse).toBe(200);
```

### Value Assertions

:::warning
These assertions do not auto-retry. Prefer [auto-retrying assertions](https://playwright.dev/docs/test-assertions#auto-retrying-assertions) whenever possible.
:::

```ts
import { expect } from '@playwright/test';

// Value is the same
const count = 5;
expect(count).toBe(5);

// Value is similar - deep equality and pattern matching
const obj = { a: 1, b: { c: 2 } };
expect(obj).toEqual({ a: 1, b: { c: 2 } });

// Value is truthy
const isActive = 'hello';
expect(isActive).toBeTruthy();

// Value is falsy
const isDisabled = 0;
expect(isDisabled).toBeFalsy();

// Array or string has length
const fruits = ['apple', 'banana', 'cherry'];
expect(fruits).toHaveLength(3);
expect('Playwright').toHaveLength(10);

// String contains a substring / Array contains element
expect('Playwright is awesome').toContain('awesome');
expect(fruits).toContain('banana');

// Array or set contains a similar element
const users = [{ id: 1 }, { id: 2 }];
expect(users).toContainEqual({ id: 2 });

// String matches a regular expression
expect('user@example.com').toMatch(/.+@example\.com/);

// Object contains specified properties
const user = { id: 1, name: 'Alice', role: 'admin' };
expect(user).toMatchObject({ name: 'Alice' });

// Object is an instance of a class
class Person {}
const person = new Person();
expect(person).toBeInstanceOf(Person);

// Function throws an error
const throwError = () => { throw new Error('Oops!'); };
expect(throwError).toThrow('Oops!');
```

### Asymmetric matchers

These expressions can be nested in other assertions to allow more relaxed matching against a given condition.

```ts
import { expect } from '@playwright/test';

// Matches any instance of a class/primitive
const str = 'hello';
expect(str).toEqual(expect.any(String));

const num = 42;
expect(num).toEqual(expect.any(Number));

// Matches anything
const value = undefined;
expect(value).toEqual(expect.anything());

// Array contains specific elements
const fruits = ['apple', 'banana', 'cherry'];
expect(fruits).toEqual(expect.arrayContaining(['banana', 'cherry']));

// Array contains elements of specific type
const numbers = [1, 2, 3];
expect(numbers).toEqual(expect.arrayOf(expect.any(Number)));

// Number is approximately equal
const pi = 3.14159;
expect(pi).toBeCloseTo(3.14, 2);

// Object contains specific properties
const user = { id: 1, name: 'Alice', role: 'admin' };
expect(user).toEqual(expect.objectContaining({ name: 'Alice' }));

// String contains a substring
const message = 'Playwright is awesome!';
expect(message).toEqual(expect.stringContaining('awesome'));

// String matches a regular expression
const email = 'user@example.com';
expect(email).toEqual(expect.stringMatching(/.+@example\.com/));
```

For the complete list, see [assertions guide](https://playwright.dev/docs/test-assertions).

### Custom expect message

```ts
import { expect } from '@playwright/test';

const value = 42;
expect(value, 'Value should be 42').toBe(42);
```

### Polling assertions

Poll a function until it returns the expected value, with configurable intervals and timeout:

```ts
// Keep calling a function until it returns the expected value
await expect.poll(async () => {
  // Example: fetch status code from an API or page
  const response = await page.request.get('/health');
  return response.status();
}, {
  // Probe, wait 1s, probe, wait 2s, probe, wait 10s, probe...
  intervals: [1_000, 2_000, 10_000],
  timeout: 60_000
}).toBe(200);
```

Retry test until it passes:

```ts
await expect(async () => {
  const response = await page.request.get('https://api.example.com');
  expect(response.status()).toBe(200);
}).toPass({
  // Probe, wait 1s, probe, wait 2s, probe, wait 10s, probe, wait 10s, probe
  // ... Defaults to [100, 250, 500, 1000].
  intervals: [1_000, 2_000, 10_000],
  timeout: 60_000
});
```

## Test Hooks

- `beforeAll`, `afterAll`: executed once, before/after all tests in its scope (root or nested describe)
- `beforeEach`, `afterEach`: executed before/after each test

```ts
import { test, expect } from '@playwright/test';

// Global hooks: run once for ALL tests in this file
test.beforeAll(() => console.log('🌍 Global beforeAll'));
test.afterAll(() => console.log('🌍 Global afterAll'));

test('Outside test #1', async ({ page }) => {
  console.log('➡️ Outside test #1');
  // ...
});

test.describe('Inside describe block', () => {
  // Local hooks: run once for tests in THIS block only
  test.beforeAll(() => console.log('📦 Local beforeAll'));
  test.afterAll(() => console.log('📦 Local afterAll'));

  test('Inside test #1', async ({ page }) => {
    console.log('➡️ Inside test #1');
    // ...
  });

  test('Inside test #2', async ({ page }) => {
    console.log('➡️ Inside test #2');
    // ...
  });
});
```

Output:

```
🌍 Global beforeAll
➡️ Outside test #1
📦 Local beforeAll
➡️ Inside test #1
➡️ Inside test #2
📦 Local afterAll
🌍 Global afterAll
```

## Annotations

```ts
import { test, expect } from '@playwright/test';

// Marks test as irrelevant; skips it
test.skip('skip this test example', async ({ page }) => {
  // ...
});

// Marks test as expected to fail; runs and verifies failure
test.fail('expected failure example', async ({ page }) => {
  // ...
});

// Marks test as failing; skips execution
test.fixme('fixme test example', async ({ page }) => {
  // ...
});

// Marks test as slow; triples the timeout
test.slow('slow test example', async ({ page }) => {
  // ...
});

// Runs only this test; skips all others
test.only('focused debugging example', async ({ page }) => {
  // ...
});

// Conditionally skip a test (or a group of tests):
test('skip this test', async ({ page, browserName }) => {
  test.skip(browserName === 'firefox', 'Still working on it');
  // ...
});
```

## Anti-patterns

### No explicit waits

Never use explicit waits in tests. Playwright's auto-waiting and retrying assertions will handle waiting for elements to be ready.

```ts
// ❌ Bad: fixed timeout — brittle and slows tests
await page.waitForTimeout(2000);

// ❌ Bad: manual wait for selector
await page.waitForSelector('button#checkout');

// ❌ Bad: waiting for element state explicitly
await page.waitForFunction(() => document.querySelector('#checkout') !== null);

// ❌ Bad: waiting for navigation manually
await page.waitForNavigation();

// ❌ Bad: waiting for load state manually
await page.waitForLoadState('networkidle');

// ❌ Bad: waiting for response manually
await page.waitForResponse('**/api/checkout');

// ❌ Bad: waiting for event manually
await page.waitForEvent('console');

// ❌ Bad: waiting for URL manually
await page.waitForURL('/checkout');
```

### No assertions in conditional blocks

Never perform assertions inside conditional blocks (e.g., `if`, `else`, `switch`):

```ts
test('login form should show error on empty submit', async ({ page }) => {
  // ❌ Bad: assertion inside conditional
  if (await page.locator('.error').isVisible()) {
    await expect(page.locator('.error')).toHaveText('Invalid Username or password');
  }
});
```

### Avoid CSS selectors

Don’t rely on styling classes or deep DOM paths.  

```ts
// ❌
page.locator('div.content > ul > li:nth-child(1)')
```

### Avoid attribute and content retrieval

Do not retrieve element's content or attribute value and perform manual assertions on them:

```ts
// ❌ Bad: getting element's content
const descriptionText = await page.getByText('welcome').textContent();

// ❌ Bad: perform assertion without auto-waiting
expect(descriptionText).toMatch(
  /[A-Z][a-z]{2},\s+\d{1,2}\s+[A-Z][a-z]{2},\s+\d{4}/
);

// ❌ Bad: getting element's attribute value
expect(await page.getByText('welcome').isVisible()).toBe(true);

// ❌ Bad: getting page URL and performing manual assertion
expect(page.url()).toContain('/dashboard');
```
