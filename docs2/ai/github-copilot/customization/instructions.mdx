# Custom instructions

## Introduction

Custom instructions are applied automatically to all chat requests or to specific files only. 

It eliminates the need for manually including context in every chat prompt.


## Types of instructions files

### A single `copilot-instructions.md` file

Automatically applies to all chat requests (in all chat sessions) in the workspace.

Stored at `.github/copilot-instructions.md`.

To generate workspace instruction file, type `/init` in chat.

Use `.github/copilot-instructions.md` when you want a consistent behavior across all different kinds of operations (code generation, code review, run tests, etc.).

Example use cases:
- Ask for user confirmation on uncertain scenarios
- Run lint and formatter after making code changes
- Do not auto-commit or auto-push


### `.instructions.md` files

Applied when files that the agent is working on match a specified pattern.

Stored in the workspace (`.github/instructions/*.instructions.md`) or user profile directory.

To generate an instructions file, type `/create-instruction` in chat.

Use when you need different rules for different file types or frameworks.

Example use cases:
- Different conventions for frontend vs. backend code
- Language-specific guidelines in a monorepo (`.ts`, `.tsx`, `.py`, etc.)
- Framework-specific patterns for specific modules (`.dto.ts`, `.controller.ts`, etc.) 
- Architecture style conventions (`ports/`, `services/`, etc.)
- Specialized rules for test files or documentation

Example structure:

```text
.github/instructions/
  frontend/
    react.instructions.md
    accessibility.instructions.md
  backend/
    python.instructions.md
  testing/
    unit-tests.instructions.md
```

Example instruction file:

```md title=".github/instructions/backend/python.instructions.md"
---
name: 'Python Standards'
description: 'Coding conventions for Python files'
applyTo: '**/*.py'
---
# Python coding standards
- Follow the PEP 8 style guide.
- Use type hints for all function signatures.
- Write docstrings for public functions.
- Use 4 spaces for indentation.
```

> [!TIPS]
> You can use Markdown links to reference specific context in instructions files


### `AGENTS.md` files

Automatically applies to all chat requests in the workspace or to specific subfolders (experimental).

Stored in the root of the workspace or in subfolders (experimental).

Use `AGENTS.md` when:
- You work with multiple AI agents in your workspace and want to apply some instructions to all of them
- You want subfolder-level instructions that apply to specific parts of a monorepo

> [!TIPS]
> For folder-specific instructions, you can also use multiple `.instructions.md` files with different `applyTo` patterns that match the folder structure.


### `CLAUDE.md` file

Automatically applies to all chat requests in the workspace.

Stored in the workspace root, `.claude` folder, or user home directory.

Use when you want a single set of instructions recognized by Claude Code or other Claude-based tools.


### Organization-level instructions

Share instructions across multiple workspaces and repositories within a GitHub organization.

Defined at the GitHub organization level.


## Manage instructions

To list/create/modify instructions, use the **Chat: Configure Instructions** command from the **Command Palette** (⇧⌘P) and select the instructions file from the Quick Pick.
